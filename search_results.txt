Podemos dizer que um Banco de Dados é uma coleção de dados persistentes, usada pelos sistemas de aplicação de uma determinada empresa. Mas qual é a ideia de persistência? Entende-se por dados persistentes aqueles que, uma vez aceitos por um SGBD (Sistema Gerenciador de Banco de Dados) para a entrada no banco de dados, somente poderão ser removidos por uma requisição explícita ao SGBD. Bancos de dados são utilizados por empresas. Chamamos de empresa qualquer organização comercial, científica, técnica ou outras. Podendo ser um único indivíduo, corporação ou grande empresa. 1 COMO GUARDAR AS INFORMAÇÕES? 1.1 Introdução A modelagem de dados permite que as informações existentes em um contexto de negócio no mundo real sejam estudadas, representadas e analisadas em algum banco de dados. A modelagem de dados é um método de análise que, a partir de fatos relevantes a um contexto de negócio, determina a perspectiva dos dados, permitindo organizá-los em estruturas bem definidas e estabelecer regras de dependência entre eles, além de produzir um modelo expresso por uma representação descritiva e gráfica. A modelagem conceitual visa o entendimento da situação tanto pelas pessoas de negócio como pelas pessoas técnicas envolvidas na solução a ser desenvolvida e implementada. O modelo mais popular e utilizado nas últimas décadas é o Modelo Entidade-Relacionamento. Mais recentemente, o Modelo de Grafos e o Modelo de Documentos têm representado muito bem os dados necessários para as soluções envolvendo as novas tecnologias como IoT e redes sociais. A modelagem lógica visa uma representação mais próxima da tecnologia a ser implementada e visa o entendimento dos profissionais destinados à implementação como desenvolvedores de software e engenheiros de dados. O Modelo Relacional baseado na teoria dos conjuntos e álgebra linear é um dos mais conhecidos por estar associado aos bancos de dados relacionais. A modelagem física é composta pelo conjunto de comandos para criação dos metadados do modelo nos bancos de dados estruturados e pelo conjunto de comandos para instanciação das ocorrências no banco de dados. A SQL é uma linguagem padrão para todos os bancos de dados relacionais. Nos bancos Not Only SQL (NoSQL) ainda não existe um padrão e cada banco tem sua linguagem. Neste capítulo detalharemos o modelo conceitual e abordaremos o modelo Entidade-Relacionamento (ER) com seus principais elementos entidades, atributos, relacionamentos, cardinalidade e normalização.

A existência pode ser física, como pessoas, casa, relógio, computadores, funcionários; ou abstrata, como um serviço, uma disciplina escolar, uma consulta médica. Entidade representa um conjunto de dados que precisa ser armazenado em um banco de dados relacional. 1.5.1.1 Reconhecimento das Entidades O reconhecimento de entidades é algo bastante fácil. Começaremos por focalizar o problema em pauta nos perguntando: "Quais são as coisas neste problema?". A maioria delas provavelmente cairá em uma das seguintes categorias: coisas tangíveis, funções, incidentes e interações.
• Coisas Tangíveis Estes são os objetos mais fáceis de serem reconhecidos no mundo real. Dado um problema apropriado, não poderíamos deixar de encontrar um objeto, como: avião, reator nuclear, cavalo de corrida, livro e veículo.

1.2 Modelo Conceitual Representa uma abstração da realidade com o objetivo de conter os dados necessários dentro do contexto de negócio observado que serão persistidos em um banco de dados. São funções de um modelo conceitual de dados: • Conhecer melhor o contexto de negócio. • Entender dos processos e regras de negócio. • Expressar as necessidades de informação da organização. • Apoiar a definição da abrangência das aplicações, delimitando seu escopo. • Apontar as necessidades, possibilitando uma melhor definição da estrutura de persistência, de modo que seja apresentada uma estrutura de armazenamento flexível, facilitando sua manutenção. • Contribuir na perspectiva da organização a respeito dos seus dados, seja unificada. É uma macrodefinição ou descrição de alto nível, pois retrata a realidade de uma organização, um processo de negócio, um setor ou um departamento, com foco no entendimento do contexto e na representação de uma realidade de uma forma simples e de fácil compreensão pelo usuário final. O modelo conceitual de dados representa as informações que existem no contexto do negócio, com maior foco nos processos. Esse modelo utiliza termos e linguagem próprios do negócio, sendo mais adequados ao dia a dia do segmento ou da área de negócio envolvidos no projeto. A técnica de modelagem conceitual mais difundida é a abordagem Entidade-Relacionamento (ER). O modelo conceitual é representado por um diagrama denominado “Diagrama Entidade-Relacionamento” (DER), ou Modelo Entidade-Relacionamento (MER).
